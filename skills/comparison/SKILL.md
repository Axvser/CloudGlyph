# Comparison — Technology Comparison & Evaluation Documentation

> **Sub-skill.** Load only when the user asks to write a technology comparison or evaluation.

## When to Load

- "Compare framework A and framework B"
- "Technology selection evaluation"
- "X vs Y comparison"
- "What are the pros and cons of this approach"

## Evaluation Dimensions

| Dimension | Description |
|---|---|
| Features | Core capability comparison |
| Performance | Benchmark data (display via KaTeX tables) |
| Usability | Learning curve, API design quality |
| Ecosystem | Community activity, plugins/extensions, documentation quality |
| Maintainability | Code quality, test coverage, architecture clarity |
| Licensing | Open-source license compatibility |

## Output Format

- **Overview table** — Score matrix across dimensions
- **Feature comparison table** — Feature-by-feature comparison
- **Performance charts** — Mermaid bar charts or KaTeX data tables
- **Scenario recommendations** — Which option is best for which scenario
- **Conclusion** — Summary and recommendation
