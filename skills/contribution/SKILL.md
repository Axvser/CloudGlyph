# Contribution — Contribution Guides & Community Collaboration Docs

> **Sub-skill.** Load only when the user asks to write a contribution guide or community documentation.

## When to Load

- "Write CONTRIBUTING.md"
- "Create a contribution guide"
- "How to contribute to this project"
- "Explain the PR workflow"

## Standard Sections

| Section | Content |
|---|---|
| Welcome | Project philosophy, code of conduct link |
| Environment Setup | Dev environment configuration steps, dependency installation |
| Branch Strategy | Git workflow, branch naming conventions |
| Code Standards | Coding style, lint configuration, naming conventions |
| Submitting a PR | PR template, review process, CI checks |
| Testing Requirements | Test coverage targets, how to run tests |
| Documentation Requirements | When documentation updates are required |
| Release Process | Version numbering, changelog maintenance |

## Format Suggestions

- Use a Mermaid flowchart to show the full PR lifecycle from submission to merge
- Use tables to list all development commands and their purposes
- Provide template code blocks that contributors can copy and use directly
